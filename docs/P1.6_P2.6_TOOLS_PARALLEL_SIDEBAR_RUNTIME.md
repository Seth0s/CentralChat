## P1.6 — Tool paralela

### Algoritmo

```go
func (a *AgentRuntime) executeTools(toolCalls []inference.ToolCall) []ToolResult {
    results := make([]ToolResult, len(toolCalls))
    phases := [][]int{
        phaseIndices(toolCalls, isReadOnly),  // fase 0: paralela
        phaseIndices(toolCalls, isShell),      // fase 1: sequencial
        phaseIndices(toolCalls, isWrite),      // fase 2: sequencial
    }
    
    for _, batch := range phases {
        if len(batch) == 0 { continue }
        if isReadOnly(toolCalls[batch[0]].Function.Name) && len(batch) > 1 {
            a.executeParallel(toolCalls, batch, results)
        } else {
            for _, i := range batch {
                results[i] = a.executeOne(toolCalls[i])
            }
        }
    }
    return results
}
```

### Classificação de tools

| Tool | Fase | Paralelizável |
|------|------|---------------|
| `read_file` | 0 (read) | ✅ Sim |
| `search_files` | 0 (read) | ✅ Sim |
| `memory` (search) | 0 (read) | ✅ Sim |
| `session_search` | 0 (read) | ✅ Sim |
| `terminal` | 1 (shell) | ❌ Sequencial |
| `write_file` | 2 (write) | ❌ Sequencial |
| `patch` | 2 (write) | ❌ Sequencial |
| `memory` (add) | 2 (write) | ❌ Sequencial |

### Semáforo

Limitar a 4 operações paralelas simultâneas para não saturar I/O:

```go
var toolSem = make(chan struct{}, 4)

func (a *AgentRuntime) executeParallel(calls []inference.ToolCall, indices []int, results []ToolResult) {
    var wg sync.WaitGroup
    for _, i := range indices {
        wg.Add(1)
        go func(idx int) {
            defer wg.Done()
            toolSem <- struct{}{}
            defer func() { <-toolSem }()
            results[idx] = a.executeOne(calls[idx])
        }(i)
    }
    wg.Wait()
}
```

### Estrutura de dados

```go
type RuntimeStatus struct {
    CPUPercent    float64
    MemUsedGB     float64
    MemTotalGB    float64
    ActiveTools   []ActiveTool
    RecentTools   []ToolResult   // últimos 5
    Background    []BgCommand
    PolicyStatus  string
    ProviderInfo  string
}

type ActiveTool struct {
    Name      string
    Args      string        // resumo: "src/main.go"
    StartTime time.Time
    Status    string        // "running" | "done" | "error"
}

type BgCommand struct {
    Command    string
    PID        int
    StartTime  time.Time
    Status     string        // "running" | "done"
}
```

### Coleta de métricas

**CPU/Memória:** Usar `github.com/shirou/gopsutil/v3` (já está no ecossistema Go, amplamente usado):

```go
import "github.com/shirou/gopsutil/v3/cpu"
import "github.com/shirou/gopsutil/v3/mem"

func collectMetrics() RuntimeStatus {
    cpuPercent, _ := cpu.Percent(0, false)  // overall CPU
    memInfo, _ := mem.VirtualMemory()
    return RuntimeStatus{
        CPUPercent: cpuPercent[0],
        MemUsedGB:  float64(memInfo.Used) / 1e9,
        MemTotalGB: float64(memInfo.Total) / 1e9,
    }
}
```

**Tools ativas:** O `AgentRuntime` já tem `OnToolStart`/`OnToolResult`. Basta acumular no `RuntimeStatus` e limpar ao fim do turno.

**Comandos background:** Quando `terminal` é executado, registrar PID e duração. O executor já tem timeout — podemos rastrear processos filhos.

### Renderização

O sidebar atual mostra workspace, model, etc. Adicionar seção "Runtime" no final:

```go
func renderRuntimeSection(st RuntimeStatus) string {
    var lines []string
    lines = append(lines, sidebarLabel("Runtime"))
    lines = append(lines, fmt.Sprintf("  CPU: %3.0f%%  MEM: %.1fGB/%.1fGB", 
        st.CPUPercent, st.MemUsedGB, st.MemTotalGB))
    
    if len(st.ActiveTools) > 0 {
        lines = append(lines, "  Active tools:")
        for _, t := range st.ActiveTools {
            icon := "⏳"
            if t.Status == "done" { icon = " ✓" }
            if t.Status == "error" { icon = " ✗" }
            elapsed := time.Since(t.StartTime).Truncate(time.Second)
            lines = append(lines, fmt.Sprintf("  %s %-14s %s  [%v]",
                icon, t.Name, t.Args, elapsed))
        }
    }
    
    if len(st.Background) > 0 {
        lines = append(lines, fmt.Sprintf("  Background (%d):", len(st.Background)))
        for _, bg := range st.Background {
            elapsed := time.Since(bg.StartTime).Truncate(time.Second)
            lines = append(lines, fmt.Sprintf("  ● %s  [%v]", bg.Command, elapsed))
        }
    }
    
    return strings.Join(lines, "\n")
}
```

## Riscos e mitigações

| Risco | Mitigação |
|-------|-----------|
| `gopsutil` falha em alguns Linux | Fallback: mostrar "cpu: n/a" em vez de crash |
| PID tracking no terminal | O `executor.Execute` não retorna PID. Precisamos wrappar: `cmd.Start()` + goroutine `cmd.Wait()` |
| Sidebar fica poluído com tools rápidas | Mostrar só tools com duração > 500ms; tools instantâneas só em "Recent" |
| Paralelismo quebra ordem dos eventos TUI | `OnToolStart`/`OnToolResult` chamados na ordem de conclusão, não de início. O TUI já lida com eventos assíncronos |
